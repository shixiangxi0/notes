# 关于作者

95 后懂架构的工程师，喜欢打乒乓球、游泳，刘慈欣的《三体》。

如果你读完这些文档应该看得出来，这是一个喜欢从第一性原理思考问题、不按常理出牌的人。

这个人提出了一套软件建模解决方案——**面向因果**。主要用于需要长期运行、持续交互、不断吸收变化的系统。

它的核心概念其实非常简洁：

- 因果层 / 感知层
- 事件 / 事实 / 裁决 / 传播

其他的新名词都是从这6个里面派生出来的概念。
