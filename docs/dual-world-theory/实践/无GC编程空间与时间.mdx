# 无 GC 编程：空间与时间

在软件工程中，内存管理通常被当作一个「空间问题」来讨论：

- 这块内存分配在哪里？
- 谁持有它的引用？
- 引用关系形成了怎样的对象图？

但如果我们换一个视角，会发现内存管理本质上是在管理**时间**：

> **一个数据在时间上何时可以消失？**

不同的语言和运行时采用了不同的策略，但是究其根本都是在用**空间关系**来推断或约束不可见的**时间关系**。

---

## 1. GC的本质

很多人对 GC 的理解始终停留在表象：

> GC 是自动释放内存。

更本质地说：

> **GC 是一个运行时算法，用空间引用关系来推断对象生命周期。**

面向对象编程语言在运行时会形成复杂的空间网络，在这个网络中，生命周期是隐式的，其核心模型可以视作：

```text
Object
  |
State
  |
Reference
  |
Other Object
```

它试图回答的问题是：

```text
这个对象未来还会不会被访问？
```

于是 GC 采用近似：

```text
Root
 ↓
引用
 ↓
对象图
```

- 从 Root 可达 → 认为未来可能使用 → **保留**
- 不可达 → 认为未来不可能使用 → **回收**

> **GC 用当前的空间可达性，近似判断未来的时间存活性。**

这就是「空间推断时间」。Java、C#、JavaScript、Golang 都是这个方向：把生命周期推断交给运行时。

---

## 2. Rust

Rust是个非常有趣的例子，它的设计哲学在于：

> **它把生命周期问题提前到编译期。**

但它没有改变基本模型。依然是：

```text
对象
  ↓
引用
  ↓
对象
```

只是显式化的增加了约束：

```text
谁拥有谁？
谁比谁活得久？
```

例如：

```rust
// Player owns Weapon
// 意味着 Weapon 的生命周期 ≤ Player 的生命周期
```

编译器检查：

```text
Player drop
   ↓
Weapon 必须先 drop
```

所以 Rust 本质上是：

> **把运行时通过 GC 推断的生命周期关系，变成程序员显式声明的生命周期约束。**

对比来看，Rust将运行时推断对象生命周期的管理方式变成了编译期通过所有权机制来显式化设计对象之间的时间关系。

这也是为什么Rust很难的原因，GC其实并没有被真正意义上“消灭“，只是被“转移“到了编译阶段，整个 Rust 程序的编写就像一个逻辑严谨的数学拓扑证明题。

本质是其实依然是在用空间来管理时间，只是变得显式化了。

---

## 3.  面向因果

在 面向因果编程 的视角下，系统的基本单元不再是对象，而是事件和事实：

```text
Event → Fact → Event
```

这提供了一种不同的时间结构。例如一次攻击：

```text
attack_requested 事件
       ↓
damage_applied 事实
       ↓
is_entity_dead 事件
```

一旦 damage_applied 被裁决并完成传播，前面的 attack_requested 就可以确定死亡；一旦 entity_dead 被消费完，中间的 damage_applied 也可以进入回收阶段。不需要扫描引用图，也不需要证明所有权链。

**因为时间边界本身就是生命周期边界。**

**当时间边界明确时，内存管理自然会得到极大简化。**

这就是双世界理论倡导的**因果必须显式化管理，让时间成为一等公民**

软件工程史上最好的解决方案往往不是那些把原问题解决得更高效，而是引入一个新的抽象层，把原问题的定义域彻底改变。

相比于它的前辈，这是一种完全不同的极简内存管理哲学：

| 机制 | 核心逻辑 | 时间关系 |
|---|---|---|
| Java、Golang | 通过对象可达性，在运行时近似推断 | 运行时隐式推断 |
| Rust | 通过所有权关系，在编译期证明 | 编译期显式约束 |
| 因果运行时 | 通过事件阶段和历史边界，直接定义 | 时间与空间并列一等公民 |

因果层的数据模型可参见[因果传播网络](/docs/dual-world-theory/causal-layer-mechanism)。

面向因果认为，软件工程中存在两种完全不同的计算本质：

时间维度：竞争、顺序、传播、生命周期、一致性。

空间维度：规模、组织、嵌套、渲染、资源管理。

这要求开发者必须明确区分“因果层“与“感知层“，因果运行时负责解决时间中的冲突，建立唯一历史；感知运行时负责在空间中展开已经确定的历史。

因果运行时处理冲突，裁决，传播，是世界的真相，代表着时间上的隐喻。

感知运行时让所有空间组件无竞态并行展开，是世界的呈现，代表着空间上的隐喻。

二者不可约，不可混淆。
