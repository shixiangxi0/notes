# About the Author

An engineer who cares deeply about software architecture, enjoys table tennis and swimming, and loves Liu Cixin's *The Three-Body Problem*.

If you have read this far, you can already see a habit of thinking from first principles rather than accepting familiar software patterns as the final word.

The central proposal here is a modeling approach called **Causality-Oriented**: a way of thinking aimed at systems that run continuously, interact constantly, and must keep absorbing change.

Its core vocabulary is deliberately simple:

- External World / Internal World
- Causal Layer / Perception Layer
- Event / Fact / Adjudication / Propagation

All other terms in this theory are derived from these six core concepts.
